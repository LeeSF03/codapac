# 2. Creating a project

This page walks you through starting a new Codapac project from scratch.

## Open the New project dialog

1. Go to the **Projects** page (click **Projects** in the top bar).
2. Click **New project** in the top-right.

A small dialog appears. It has everything you need on one screen — no wizard,
no long forms.

## Fill in the fields

Only the first field is required. The rest help your squad do better work.

1. **Name** — what you'll call your project. Examples: *Acme Landing*,
   *Book Club App*, *Internal Feedback Tool*.
2. **URL slug** — auto-generated from your name (e.g. `acme-landing`).
   You can edit it. This becomes part of the project's address:
   `codapac.app/project/your-slug`.
3. **Description** (optional) — one or two sentences about what this project
   is for. Your squad reads this, so clearer is better.
   - Good: *"A marketing site for Acme's new pricing page, aimed at SMB buyers."*
   - Vague: *"A website."*
4. **Emoji and color** — pick an emoji and a color. Purely visual, but it
   makes projects easy to spot in your list.
5. **Repo link** (optional) — if you already have a code repository, paste
   its URL here. You can leave this blank for brand new projects — Codapac
   will still build and preview the project without it.

## Create it

1. Click **Create project**.
2. Codapac sets up your workspace and takes you straight to it.

That's the whole setup. No installs, no configuration.

## What happens right after you click Create

You'll land on your **project workspace** (the next guide explains every
piece of it). In the background:

- Your project is saved to your account.
- Your agent squad is attached to this project.
- The **project chat** is ready for your first message.
- The **board** starts empty, waiting for your first task.

If you typed a description, the squad has already read it — you can say
things like *"start with what I described above"* in the chat.

## Naming tips

- **Name like a product, not a ticket.** *Acme Landing* reads better than
  *fix-homepage-q2*.
- **Keep slugs short.** `acme-landing` is easier to share than
  `acme-marketing-site-v3-final`.
- **Emojis help a lot.** Pick something you'll recognize at a glance in the
  Projects list.

## If something goes wrong

- **"Slug already in use."** Someone (maybe you, earlier) used that slug.
  Edit it to something unique.
- **Create button stays greyed out.** You probably need a **name** — it's
  the only required field.
- **Wrong emoji or color?** Open the project, then [edit its settings from
  the project page](./03-project-workspace.md) — nothing you pick now is
  permanent.
