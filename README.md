# Codapac — User Guide

Welcome. This is the friendly, no-jargon guide to using Codapac.

Codapac is a workspace where you turn ideas into real software — by talking to
your **agent squad**, watching the work appear as cards on a **board**, and
opening a **live preview** of what got built. You don't need to write code.

If you can describe what you want in plain English, you can ship with Codapac.

## Who this guide is for

- Founders turning a pitch into a prototype
- Product managers shaping features and shipping them
- Students and hackathon teams who want to move fast
- Non-technical teammates who want to contribute without waiting for a
  developer to translate

## Tips before you start

- Everything you do is **saved automatically**. You can close the tab and come back.
- If something feels stuck, the [FAQ](./10-faq.md) is the fastest place to check first.
- When in doubt, **ask your squad in the project chat** — that's what they're there for.
